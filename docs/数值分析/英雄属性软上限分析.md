# 英雄属性软上限分析

> 分析日期: 2026-04-22
> 分析范围: 所有战斗属性的上限机制（软上限/硬上限/无上限）

---

## 一、属性来源总览

英雄战斗属性来自 **6 个来源**：

| 来源 | 提供的属性 | 备注 |
|------|-----------|------|
| 英雄基础成长 | atk, spd | 二级属性（critRate/critDmg/armorPen等）基础值和成长均为 0 |
| 装备（4件套） | atk(武器), dmgBonus(铠甲), critDmg(头盔), elemDmg(战马) | `statBase × level × tierMult` |
| 装备淬炼（5孔） | 13种属性（critRate/critDmg/armorPen/dmgBonus等） | 红色满级(Lv4000)后开启 |
| 符文（3槽） | 基础词条 + 特殊词条 | 每符文最多4词条 |
| 符文套装 | critRate/dmgBonus/armorPen/atk_pct/spd_pct | 2件套/3件套效果 |
| 技能/羁绊 | critRate/critDmg/atk等 | 特定英雄被动 + 阵营羁绊 |

## 二、各属性上限机制一览

| 属性 | 上限机制 | 阈值/参数 | 是否合理 |
|------|---------|----------|---------|
| **dmgBonus** (伤害加成) | 对数软上限 | threshold=0.80, scale=1.0 | ⚠️ 见分析 |
| **critDmg** (暴击伤害) | 对数软上限 | threshold=1.00, scale=1.5 | ⚠️ 见分析 |
| **elemDmg** (元素伤害) | 对数软上限 | threshold=0.80, scale=1.0 | ⚠️ 见分析 |
| **critRate** (暴击率) | **无软上限** | 仅受来源数量限制 | ⚠️ 潜在问题 |
| **armorPen** (穿甲) | **无软上限** | 仅受来源数量限制 | ⚠️ 潜在问题 |
| **spd** (攻速加成) | 硬上限 | SPD_BONUS_MAX = 0.30 (30%) | ✅ 合理 |
| **atk** (攻击力) | 无上限 | 仅受等级/进阶/升星/装备上限限制 | ✅ 被HP缩放平衡 |

## 三、软上限公式

```
SoftCapStat(raw, cap):
    if raw <= cap.threshold:
        return raw                              -- 阈值内：线性（1:1）
    else:
        over = raw - cap.threshold
        return cap.threshold + cap.scale × ln(1 + over / cap.scale)   -- 超出：对数衰减
```

**特性**：
- 阈值以下：投入产出 1:1
- 阈值以上：边际收益递减（对数衰减）
- `scale` 越大，衰减越慢（critDmg的scale=1.5 比 dmgBonus的scale=1.0 衰减更慢）

## 五、无软上限属性分析

### 5.1 critRate（暴击率） — 无软上限 ⚠️

**来源堆叠上限**：

| 来源 | 最大值 | 条件 |
|------|-------|------|
| 英雄基础 | 0 | 所有英雄 critRate 基础和成长均为 0 |
| 装备主属性 | 0 | 无装备槽提供 critRate |
| 淬炼（5孔） | 5 × 0.20 = **1.00** | 每孔 maxValue=0.20，红色档1.50倍=0.30，但受maxValue限制 |
| 符文词条（3×4词条） | 12 × 0.10 × 1.35 = **1.62** | 极端假设全红品质全选critRate |
| 符文套装(雷霆2件) | **0.10** | thunder set2 |
| 阵营羁绊(人类4) | **0.08** | human_alliance |
| 跨阵营羁绊(死亡军团) | **0.10** | undead2 + demon2 |
| 英雄技能(深渊猎手) | **0.15** | deadly_crossbow 被动 |
| 英雄技能(永恒魔皇) | **0.20** | archfiend_strike 被动 |
| 英雄光环(堕天使) | **0.12** | fallen_glory 光环 |

**理论极限**：淬炼1.00 + 符文~0.40(现实) + 套装0.10 + 羁绊0.18 + 技能0.20 ≈ **1.88** (188%)

**现实极限**（考虑词条随机性）：淬炼0.60 + 符文0.20 + 套装0.10 + 羁绊0.08 + 技能0.15 ≈ **1.13** (113%)

**问题**：critRate > 100% 意味着必定暴击（多余的完全浪费），但代码中没有对 critRate 做 `min(1.0)` 限制。虽然超过100%在数学上不影响伤害（判定 `rand < critRate` 即可），但玩家会觉得投入被浪费。

### 5.2 armorPen（穿甲） — 无软上限 ⚠️

**来源堆叠上限**：

| 来源 | 最大值 | 条件 |
|------|-------|------|
| 英雄基础 | 0 | 所有英雄 armorPen 基础和成长均为 0 |
| 装备主属性 | 0 | 无装备槽提供 armorPen |
| 淬炼（5孔） | 5 × 0.20 = **1.00** | 每孔 maxValue=0.20 |
| 符文词条（3×4词条） | 12 × 0.10 × 1.35 = **1.62** | 极端全选 armorPen |
| 符文套装(亡灵2件) | **0.08** | undead set2 |
| 英雄技能 | 0 | 无英雄提供穿甲 |

**理论极限**：淬炼1.00 + 符文~0.40(现实) + 套装0.08 ≈ **1.48** (148%)

**现实极限**：淬炼0.50 + 符文0.18 + 套装0.08 ≈ **0.76** (76%)

**问题**：armorPen > 100% 意味着"负护甲"（增伤），且在终局被 ENEMY_SCALING.armorPenResist 削减30%，即有效穿甲约 0.76×0.70 = 0.53。虽然不太容易堆到离谱的值，但缺乏软上限意味着极限BD（Build）玩家可以完全忽略护甲机制。

## 六、敌人防御缩放 vs 英雄软上限后属性

终局（关卡6000）时，双方数值对比：

| 属性 | 英雄软上限后 (装备+淬炼+符文) | 敌人减免 | 最终有效值 |
|------|---------------------------|---------|----------|
| dmgBonus | ~4.49 + 0.50 ≈ **5.0** (500%) | dmgBonusReduce=0.35 → ×0.65 | **3.25** (325%) |
| critDmg | ~6.55 + 0.80 ≈ **7.35** (735%) | critDmgReduce=0.40 → ×0.60 | **4.41** (441%) |

## 七、软上限参数评估

### 7.1 dmgBonus 软上限

| 指标 | 值 | 评价 |
|------|-----|------|
| 阈值 | 0.80 (80%) | **偏低** — 蓝色装备Lv400就触达 |
| scale | 1.0 | 衰减速度适中 |
| 红色满级有效率 | 11.2% | 🔴 **极低** — 装备升级在中后期失去意义 |

**问题**：阈值太低，玩家从蓝色品质开始就感受到衰减，紫色品质时有效率已降至44%，升级装备的正反馈急剧减弱。

### 7.2 critDmg 软上限

| 指标 | 值 | 评价 |
|------|-----|------|
| 阈值 | 1.00 (100%) | 中等 — 绿色Lv333触达 |
| scale | 1.5 | 衰减较慢（比dmgBonus宽松） |
| 红色满级有效率 | 10.9% | 🔴 **极低** |

**问题**：虽然 scale=1.5 让衰减比 dmgBonus 慢，但因为 critDmg 基础值成长更快（0.003/级 vs 0.002/级），最终有效率几乎一样低。

### 7.3 elemDmg 软上限

| 指标 | 值 | 评价 |
|------|-----|------|
| 阈值 | 0.80 (80%) | 与 dmgBonus 相同 |
| scale | 1.0 | 与 dmgBonus 相同 |
| 红色满级有效率 | 11.2% | 🔴 **极低** |

**问题**：元素伤害还要额外被 `THEME_ELEMENT_RESIST` 和 `elemDmgReduce` 双重削减，有效收益更低。

## 八、核心问题诊断

### 问题 1: 装备升级在中后期失去正反馈 🔴

**现象**：从蓝色品质（约Lv500）开始，每次装备升级的边际收益快速下降。到紫色品质，玩家每升100级装备只获得原始值的44%→36%→...。

**根因**：阈值太低，而装备等级跨度太大（1→4000）。阈值仅在 Lv400（绿色末期）就被突破。

**影响**：玩家在游戏中期就感觉"装备升了白升"，付费动力下降。

### 问题 2: critRate 可超过 100%，投入浪费 ⚠️

**现象**：极限BD下 critRate 可达 113%+，超过100%的部分完全浪费。

**建议**：在代码中添加 `critRate = math.min(critRate, 1.0)` 的硬上限，或者添加类似 dmgBonus 的软上限（如 threshold=0.60, scale=0.3），让高暴击率的边际收益递减但永远有正收益。

### 问题 3: armorPen 缺乏递减机制 ⚠️

**现象**：穿甲无上限，极端情况下可完全无视护甲甚至产生"负护甲增伤"。

**建议**：添加软上限（如 threshold=0.50, scale=0.5），或硬上限为1.0（最多100%穿甲=真伤）。

### 问题 4: 三个软上限属性有效率趋同 ⚠️

**现象**：dmgBonus/critDmg/elemDmg 在终局的有效率都约 11%，没有差异化。

**建议**：考虑差异化设计，例如让 elemDmg 的阈值更高（鼓励元素流派），或让 critDmg 的 scale 更大（鼓励暴击流派），制造 BD 多样性。

### 问题 5: 双重削减叠加过重 ⚠️

**现象**：软上限 + ENEMY_SCALING 减免同时作用。以 dmgBonus 为例：
- 原始值 40.0 → 软上限后 4.49 → 被 dmgBonusReduce 0.35 削减 → 最终 2.92

这意味着玩家投入的 4000% 伤害加成，最终只剩 292%，有效率仅 7.3%。

**建议**：如果保留 ENEMY_SCALING 减免，应提高软上限阈值；或者弱化 ENEMY_SCALING，让软上限独立承担平衡压力。

## 九、SPD（攻速）硬上限评估

| 指标 | 值 |
|------|-----|
| 硬上限 | 0.30 (30%) |
| 曲线 | 分段线性：1→0%, 3→10%, 10→20%, 50→27%, 200→30% |
| 前期收益 | 高（倍率1→3 获得 10%） |
| 后期收益 | 极低（倍率50→200 仅获得 3%） |

**评价**：✅ 硬上限30%设计合理，分段递减曲线避免了攻速失控。攻速是乘法系数，无上限会导致DPS无限增长。

## 十、优化建议

### 建议 1: 提高软上限阈值

```lua
-- 当前值（过低）
Balance.SOFT_CAPS = {
    dmgBonus = { threshold = 0.80, scale = 1.0 },
    critDmg  = { threshold = 1.00, scale = 1.5 },
    elemDmg  = { threshold = 0.80, scale = 1.0 },
}

-- 建议值（使紫色品质仍有良好收益，橙色品质开始明显衰减）
Balance.SOFT_CAPS = {
    dmgBonus = { threshold = 5.00, scale = 2.0 },   -- 紫Lv2000=8.0, 软上限后=6.10 (76%)
    critDmg  = { threshold = 8.00, scale = 3.0 },   -- 紫Lv2000=12.0, 软上限后=9.89 (82%)
    elemDmg  = { threshold = 5.00, scale = 2.0 },   -- 同 dmgBonus
}
```

**效果**：让装备升级在紫色品质仍保持 75%+ 有效率，橙色品质 50%+，红色品质 30%+。

### 建议 2: 为 critRate 添加上限

```lua
-- 方案 A：硬上限（简洁）
tower.critRate = math.min(tower.critRate, 1.0)

-- 方案 B：软上限（更柔和，鼓励适度投入）
Balance.SOFT_CAPS.critRate = { threshold = 0.50, scale = 0.3 }
-- 堆到0.80原始值 → 软上限后0.50 + 0.3×ln(2.0) = 0.71
-- 堆到1.13原始值 → 软上限后0.50 + 0.3×ln(3.1) = 0.84
```

### 建议 3: 为 armorPen 添加上限

```lua
-- 穿甲硬上限（最多100%穿甲 = 真实伤害，不出现负护甲）
Balance.SOFT_CAPS.armorPen = { threshold = 0.40, scale = 0.4 }
-- 或简单硬上限
tower.armorPen = math.min(tower.armorPen, 1.0)
```

### 建议 4: 差异化三个软上限属性

| 属性 | 建议定位 | 阈值 | scale | 设计理念 |
|------|---------|------|-------|---------|
| dmgBonus | 通用增伤 | 5.0 | 2.0 | 稳定收益，适合所有流派 |
| critDmg | 爆发流 | 8.0 | 3.0 | 高阈值+慢衰减，鼓励暴击BD |
| elemDmg | 元素流 | 6.0 | 2.5 | 中等阈值，配合元素克制更有优势 |

### 建议 5: 软上限与 ENEMY_SCALING 二选一

两套递减机制叠加导致终局属性有效率过低（7-11%）。建议：

- **方案A**：保留软上限，弱化 ENEMY_SCALING（各项减免降低50%）
- **方案B**：移除软上限，仅保留 ENEMY_SCALING 作为唯一递减手段
- **方案C（推荐）**：保留两者但大幅提高软上限阈值（如建议1），使软上限只在极端堆叠时生效
