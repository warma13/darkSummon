# 序列帧与头像规范

## 一、序列帧 (Spritesheet)

### 1.1 基本格式

| 项目 | 规范 |
|------|------|
| 格式 | PNG，透明背景 |
| 排列方式 | **横排等宽**，所有帧在同一行 |
| 帧数 | 通常 3 帧（idle / attack / projectile），近战英雄可 2 帧（无弹体） |
| 总宽度 | 帧宽 × 帧数 |
| 总高度 | 帧高（单行） |

### 1.2 尺寸规格

| 级别 | 帧尺寸 | 总尺寸 (3帧) | 适用对象 |
|------|--------|-------------|---------|
| **标准** | 128 × 163 px | 384 × 163 px | N / R / SR / SSR / UR 英雄 |
| **高清** | 256 × 326 px | 768 × 326 px | 主角、BOSS、特殊 UR/LR 英雄 |

> 高清规格为标准的 2 倍，保持相同宽高比 (0.79:1)

### 1.3 帧内容定义

| 帧索引 | 内容 | 说明 |
|--------|------|------|
| 0 | **idle** (待机) | 默认展示姿态，同时作为 UI 头像来源 |
| 1 | **attack** (攻击) | 攻击动作，攻击动画时切换 |
| 2 | **projectile** (弹体) | 远程攻击的弹体图标。近战英雄可省略此帧 (cols=2) |

### 1.4 绘制要求

- 角色居中放置在帧内，左右留适当边距
- 角色**默认朝右**，游戏内通过 flipX 处理朝左
- 同一英雄的 3 帧保持一致的角色大小和位置基线
- idle 帧应展示角色最具辨识度的姿态（用于头像裁切）

### 1.5 命名规则

```
assets/image/{英雄英文ID}_sprite.png      # 标准英雄
assets/image/{英雄英文ID}_spritesheet.png  # 高清/特殊英雄
```

示例：
```
image/skeleton_archer_sprite.png       # 骷髅弓手 (128×163, 3帧)
image/shadow_lord_spritesheet.png      # 暗影君主 (256×326, 3帧)
```

## 二、头像 (Avatar)

### 2.1 来源

头像**统一从序列帧第一帧 (idle) 自动裁切生成**，不单独制作。

### 2.2 生成方式

```bash
python3 tools/crop_avatars.py          # 只处理缺失的头像
python3 tools/crop_avatars.py --force  # 强制全部重新生成
```

### 2.3 输出

| 项目 | 规范 |
|------|------|
| 格式 | PNG，保留透明通道 |
| 尺寸 | 与序列帧第一帧一致（标准 128×163，高清 256×326） |
| 路径 | `assets/image/avatars/avatar_{icon}.png` |

---

## 三、新增英雄流程

```
1. 生成序列帧图片 (AI 生图或美术制作)
   → 遵循上方尺寸规格和帧内容定义
   → 保存到 assets/image/{id}_sprite.png

2. 注册到代码
   → 在 scripts/Game/Renderer_Utils.lua 中添加 SpriteSheet.Register

3. 自动生成头像
   → 运行 python3 tools/crop_avatars.py
   → 自动裁切第一帧到 assets/image/avatars/

4. 完成，无需手动裁切
```

## 五、AI 生图提示词模板

生成序列帧时使用以下提示词结构：

**标准规格 (128×163, 3帧)**：
```
游戏角色像素风格序列帧，横排3帧，透明背景，
帧1-站立待机姿态，帧2-攻击动作，帧3-弹体/技能特效，
{角色描述}，统一角色大小，角色朝右。
```

**高清规格 (256×326, 3帧)**：
```
高品质游戏角色序列帧，横排3帧，透明背景，
帧1-站立待机姿态，帧2-攻击动作，帧3-弹体/技能特效，
{角色描述}，统一角色大小，角色朝右，精致细节。
```
