## 二、问题分级

### P0 -- 严重瓶颈（建议立即优化）

#### 2.1 减益粒子系统：渲染热点 + GC 压力源

**文件**: `Renderer_Utils.lua` — `SpawnParticle` / `DrawEnemyDebuffParticles` / `UpdateDebuffParticles`

**问题**:
1. 每个减益粒子需要 7 次 NanoVG 调用（save/translate/rotate/imagePattern/rect/fill/restore）
2. 20 个带减益的敌人 × 30 粒子上限 = 600 个粒子 = **4200 次 NanoVG 调用/帧**，占总渲染量的 40%~70%
3. 每个粒子是独立的 Lua table（10 个字段），创建后存活 0.4~1.2 秒即被 GC 回收
4. 高峰期每秒创建/销毁约 600 个临时 table，造成 GC 抖动

**优化方案**:

```
方案 A: 对象池（推荐，改动最小）
──────────────────────────────
-- 模块顶部定义池
local particlePool = {}
local poolSize = 0

local function AcquireParticle()
    if poolSize > 0 then
        local p = particlePool[poolSize]
        particlePool[poolSize] = nil
        poolSize = poolSize - 1
        return p
    end
    return {}  -- 池空时才分配
end

local function ReleaseParticle(p)
    poolSize = poolSize + 1
    particlePool[poolSize] = p
end

-- SpawnParticle 中复用而非新建
local function SpawnParticle(texKey, x, y, vx, vy, life, pSize, rot, rotSpd)
    local p = AcquireParticle()
    p.tex = texKey; p.x = x; p.y = y
    p.vx = vx; p.vy = vy
    p.life = life; p.maxLife = life
    p.size = pSize; p.rot = rot or 0; p.rotSpd = rotSpd or 0
    return p
end

-- UpdateDebuffParticles 中归还而非置 nil
if p.life <= 0 then
    ReleaseParticle(pool[i])
    pool[i] = pool[#pool]
    pool[#pool] = nil
    ...
end

预期收益: 消除 ~600 次/秒 table 分配，减少 GC 暂停
```

```
方案 B: 降低粒子密度（效果与性能的权衡）
──────────────────────────────
-- 调整常量
MAX_DEBUFF_PARTICLES_PER_ENEMY = 15  -- 从 30 降到 15
-- 降低各减益的生成概率（dt 系数减半）
-- 例如 ice: dt * 4（从 dt * 8）

预期收益: NVG 调用量直接减半（4200 → 2100）
```

```
方案 C: 批量渲染（深度优化）
──────────────────────────────
-- 按纹理分组，一次 nvgSave/nvgRestore 绘制同类粒子
-- 减少状态切换开销
local groups = {}
for eid, pool in pairs(debuffParticles) do
    for _, p in ipairs(pool) do
        local g = groups[p.tex] or {}
        g[#g + 1] = p
        groups[p.tex] = g
    end
end
for tex, ps in pairs(groups) do
    nvgSave(vg)
    -- 设置纹理一次
    for _, p in ipairs(ps) do
        -- 仅做 translate + draw，省去重复的纹理设置
    end
    nvgRestore(vg)
end

预期收益: 减少 nvgSave/nvgRestore 对数，约省 30%~40% NVG 调用
```

**建议**: 先实施方案 A（对象池）+ 方案 B（降低密度），效果不足时再加方案 C。

#### 2.2 目标搜索：O(T×E) 暴力扫描

**文件**: `Combat.lua` — `FindTarget`

**问题**:
- 每帧为每个塔遍历全部敌人寻找最近目标
- 20 塔 × 100 敌人 = 2000 次距离计算/帧
- 链弹 `FindChainTarget` 每次弹跳再扫描全部敌人

**优化方案**:

```
方案 A: 空间网格哈希（推荐）
──────────────────────────────
-- 利用现有 Grid 系统的格子大小（CELL_SIZE）做空间分区

local spatialGrid = {}
local SPATIAL_CELL = Config.CELL_SIZE  -- 复用现有格子尺寸

function RebuildSpatialGrid()
    -- 每帧开头重建（O(E)）
    spatialGrid = {}
    for _, e in ipairs(State.enemies) do
        if e.alive then
            local cx = math.floor(e.x / SPATIAL_CELL)
            local cy = math.floor(e.y / SPATIAL_CELL)
            local key = cx * 10000 + cy
            local cell = spatialGrid[key]
            if not cell then
                cell = {}
                spatialGrid[key] = cell
            end
            cell[#cell + 1] = e
        end
    end
end

function FindTargetFast(tower, tx, ty, range)
    local cx = math.floor(tx / SPATIAL_CELL)
    local cy = math.floor(ty / SPATIAL_CELL)
    local r = math.ceil(range / SPATIAL_CELL)
    local best, bestDist = nil, range * range
    for dx = -r, r do
        for dy = -r, r do
            local key = (cx + dx) * 10000 + (cy + dy)
            local cell = spatialGrid[key]
            if cell then
                for _, e in ipairs(cell) do
                    if e.alive then
                        local ddx = e.x - tx
                        local ddy = e.y - ty
                        local d2 = ddx * ddx + ddy * ddy
                        if d2 < bestDist then
                            bestDist = d2
                            best = e
                        end
                    end
                end
            end
        end
    end
    return best
end

预期收益: FindTarget 从 O(T×E) 降至 O(T×K)，K 为邻近格内敌人数
         20 塔 × 100 敌人 → 20 塔 × ~15 敌人 = 300 次/帧（降 85%）
```

```
方案 B: 排序 + 跳过（轻量替代）
──────────────────────────────
-- 敌人按 progress 排序，塔只检查 progress 范围内的敌人
-- 适用于环形路径上"就近攻击"的场景

预期收益: 约降 50% 迭代量，但不如空间哈希通用
```

#### 2.3 CalcFinalDamage 每次命中分配临时 table

**文件**: `Combat.lua` — `CalcFinalDamage`

**问题**:
- 每次伤害计算创建一个 `zones` table（3~7 个条目）
- AOE 技能命中 10 个敌人 = 10 个临时 table
- 高频战斗（快速攻击 + AOE）时每秒产生数百个短命 table

**优化方案**:

```lua
-- 模块顶部预分配一个可复用 table
local _zones = {}

function CalcFinalDamage(baseDamage, target, tower)
    -- 清空而非新建
    for k in pairs(_zones) do _zones[k] = nil end

    _zones.def = armorMult
    _zones.crit = critMult
    -- ... 其余字段

    local finalDmg = baseDamage
    for _, mult in pairs(_zones) do
        finalDmg = finalDmg * mult
    end
    return finalDmg, isCrit
end

预期收益: 消除每次命中的 table 分配，GC 压力显著降低
```

#### 2.5 网格逐格绘制：99 次 NVG 调用可合并

**文件**: `Renderer_Towers.lua` — `DrawGrid`

**问题**: 16 个网格格子各自执行 beginPath + roundedRect + fill + stroke = 6 次调用，共 ~96 次

**优化方案**:

```lua
-- 合并所有格子背景为一次 path
nvgBeginPath(vg)
for c = 1, Config.GRID_COLS do
    for r = 1, Config.GRID_ROWS do
        if not Grid.IsPathCell(c, r) then
            nvgRoundedRect(vg, x, y, w, h, radius)
        end
    end
end
nvgFillColor(vg, bgColor)
nvgFill(vg)

-- 合并所有格子边框为一次 stroke
nvgBeginPath(vg)
for c = 1, Config.GRID_COLS do
    for r = 1, Config.GRID_ROWS do
        if not Grid.IsPathCell(c, r) then
            nvgRoundedRect(vg, x, y, w, h, radius)
        end
    end
end
nvgStrokeWidth(vg, borderWidth)
nvgStrokeColor(vg, borderColor)
nvgStroke(vg)

预期收益: ~96 次 NVG 调用 → ~8 次（降 90%+）
```

## 五、实施路线图

### 第一阶段：低成本高收益

改动量小、风险低、效果显著的优化：

1. **减益粒子对象池** — `Renderer_Utils.lua` 改 SpawnParticle/UpdateDebuffParticles
2. **zones table 复用** — `Combat.lua` 改 CalcFinalDamage
3. **GetAliveCount 运行计数** — `Enemy.lua` + `State.lua`
4. **飘字颜色预定义** — `HeroSkills.lua` + `Combat.lua` 顶部加常量
5. **sqrt → 平方距离** — `HeroSkills.lua` 改 CheckCritSplash
6. **网格合并绘制** — `Renderer_Towers.lua` 改 DrawGrid
7. **debuffAffixes 预计算** — `Enemy.lua` 改 CreateBase + 词缀应用

预期效果：
- GC 压力降低 60%~80%
- NVG 调用减少 ~2200/帧（高压场景）
- 帧时间改善约 15%~25%

### 第二阶段：架构级优化

需要更大改动、更多测试：

1. **空间网格哈希** — `Combat.lua` 新增 RebuildSpatialGrid / FindTargetFast
2. **减益粒子批量渲染** — `Renderer_Utils.lua` 按纹理分组
3. **减益粒子密度调优** — 调整生成概率，配合视觉效果微调
4. **WorldBoss 直接引用** — `WorldBossSkills.lua` + `State.lua`
5. **生怪队列索引替代** — `BattleManager.lua` / `Wave.lua`

预期效果：
- FindTarget 性能提升 80%~90%
- NVG 调用再减少 ~1700/帧
- 帧时间进一步改善 10%~15%

### 第三阶段：长期改进

1. **属性计算 Memoization** — `HeroData.lua`
2. **BuildEnemyDef 缓存** — `Config_Enemies.lua`
3. **Grid key 数值化** — `Grid.lua`
4. **CompactArrays dirty flag** — `State.lua`
5. **粒子物理帧率无关** — `Renderer.lua`

## 七、度量与验证

### 关键度量指标

建议在优化前后用以下方式度量效果：

```lua
-- 简单的帧时间统计（加在 main.lua HandleUpdate 中）
local _frameCount = 0
local _totalDt = 0
local _maxDt = 0

function LogFrameStats(dt)
    _frameCount = _frameCount + 1
    _totalDt = _totalDt + dt
    if dt > _maxDt then _maxDt = dt end
    if _frameCount % 300 == 0 then  -- 每 5 秒输出一次
        local avgMs = (_totalDt / _frameCount) * 1000
        local maxMs = _maxDt * 1000
        print(string.format(
            "[Perf] 平均帧时: %.1fms (%.0f FPS) | 最大帧时: %.1fms | 敌人: %d | 塔: %d | 投射物: %d",
            avgMs, 1000 / avgMs, maxMs,
            #State.enemies, #State.towers, #State.projectiles
        ))
        _frameCount = 0
        _totalDt = 0
        _maxDt = 0
    end
end
```
