# 系统设计：深渊金库

> **暗影召唤师 · 次留/长留钩子系统**
> 版本 v2.0 | 2026-04-18

---

## 一、设计目标

| 目标 | 具体期望 |
|------|---------|
| **提升次日留存（D2）** | 玩家结束当天游戏前明确知道"明天有东西等我来拿" |
| **提升长期留存** | 每日"存-等-取-再存"形成低摩擦操作习惯 |
| **鼓励主动操作** | 边际递减让"分散多天操作"优于"堆满不管" |
| **不破坏数值平衡** | 产出有天花板，是现有暗影精粹来源的合理补充 |

---

## 二、货币选型：暗影精粹

### 为什么不用冥晶

冥晶含二次项（`stageQuadratic = 0.004`），第 10 关后单局产出 5,000+，日收入轻松数万，任何固定存入上限快速变得无意义，**无法做成可持续钩子**。

### 为什么选暗影精粹

| 维度 | 情况 |
|------|------|
| **日常免费产出** | ~42 精粹/天（日任 30 + 钻石宝箱 ~5 + 签到均摊 ~7） |
| **消耗方向** | SSR碎片 30、觉醒材料 40、UR碎片 50、LR碎片 100 |
| **稀缺感** | 玩家始终想要更多（碎片需求无上限），储蓄产出永不过时 |
| **不随关卡膨胀** | 产出来源固定，不受怪物掉落公式影响，适合做定量储蓄 |

暗影精粹是玩家**最有情感投入**的货币（直接关系到抽到想要英雄的碎片），用它做储蓄的"存取决策"最有感知。

## 三、核心机制

档位由**存入天数**决定，而非存入金额。第一天存享最高收益，次日可追加存入，但收益率逐档递减。**"领取并重投"重置回第一档**，形成每日登录闭环。

```
第1天存入 → 第一档（15%）→ 次日领取利息 + 追加第二档（8%）→ 再次日追加第三档（4%）
                                    ↑
                             或：全部领取并重投 → 重回第一档（15%）
```

## 四、数值设计

### 4.1 档位结构

| 档位 | 每次存入上限 | 日收益率 | 日最高产出 | 解锁条件 |
|------|-----------|---------|----------|---------|
| **第一档** | 2,000 精粹 | **15%** | 300/天 | 随时（首次存入 / 领取并重投后）|
| **第二档** | 1,000 精粹 | **8%**  | 80/天  | 第一档存入满 1 天后解锁 |
| **第三档** | 500 精粹   | **4%**  | 20/天  | 第二档存入满 1 天后解锁 |

- **三档全满时总存入**：3,500 精粹
- **三档全满时日产出**：400 精粹/天
- **"领取并重投"后**：全部归零，次日重回第一档

### 4.2 档位解锁时序

```
Day 1  ──► 存入第一档（最多 2000）
Day 2  ──► 第二档解锁，可追加存入（最多 1000）
             同时领取 Day 1 利息
Day 3  ──► 第三档解锁，可追加存入（最多 500）
             同时领取 Days 1+2 利息
Day 4+ ──► 三档齐满，每日领取 400 精粹
             或：随时"领取并重投" → 重置回 Day 1
```

### 4.3 策略对比（玩家自己选）

| 策略 | 操作 | 日产出 | 总投入 | ROI |
|------|------|--------|--------|-----|
| **每日重投** | 每天领取所有利息 + 重投第一档 | 300/天 | 2,000 精粹 | **15%** |
| **逐档累积** | 连续 3 天追加存入，之后只领息 | 400/天 | 3,500 精粹 | 11.4% |
| **轻度参与** | 只存第一档，偶尔领取 | 最高 300/天 | ≤ 2,000 精粹 | 15% |

> **核心张力**：每日重投的 ROI 最高（15%），但绝对产出低于三档全满（300 vs 400）。
> 玩家每天面临真实决策：**"今天重投拿高利率，还是追加第二档拿更多绝对量？"**
> 这个决策让金库从"摆设"变成每日都要思考的系统。

### 4.4 感知校验

| 参照 | 数值 | 说明 |
|------|------|------|
| 日常免费产出 | ~42 精粹/天 | 基准 |
| 第一档满仓日产 | 300 精粹/天 | 日常产出的 **7 倍**，长线目标明确 |
| 三档全满日产 | 400 精粹/天 | 近 10 倍日常产出，重度玩家核心收益 |
| SSR 碎片价格 | 30 精粹 | 三档全满时每天可换 13 个 SSR 碎片 |
| 存入 100 精粹 | 15 精粹/天 | 轻度参与立刻有感知 |

### 4.3 利息累计规则

| 情况 | 行为 |
|------|------|
| 当天存入，次日结算 | 利息挂账，主界面红点提醒 |
| 连续不登录 | 利息**最多积累 2 天**，之后停止增长（本金安全）|
| 超过 2 天未登录 | 提示"金库满溢，增值已暂停" |

## 五、每日闭环

### 5.1 档位解锁节奏（时序视图）

```
═══ Day 1 ═══════════════════════════════════════════════
登录 → 存入第一档（≤2000 精粹）→ 退出等待
         ↓ 次日红点提醒

═══ Day 2 ═══════════════════════════════════════════════
登录 → 金库红点 → 打开面板
   └─ 领取第一档利息（最高 300）
   └─ 【第二档已解锁！】追加存入（≤1000）
   决策：今天"重投第一档拿 15%"还是"追加第二档拿绝对量"？

═══ Day 3 ═══════════════════════════════════════════════
登录 → 领取 Day1+Day2 利息
   └─ 【第三档已解锁！】追加存入（≤500）
   三档全满后每日领取 400 精粹

═══ Day 4+ ══════════════════════════════════════════════
三档齐满，每日领息 400
   或：随时点击 [领取并重投 ▶] → 清空所有档位 → 回到 Day 1
```

### 5.2 操作流程（单次登录）

```
打开金库面板
    ↓
有待领利息？→ 领取（或选择重投）
    ↓
有新档位解锁？→ 提示追加存入
    ↓
退出，主界面显示"明日预计 +X 精粹"
```

## 七、边界情况

| 情况 | 处理 |
|------|------|
| 第一档未存 | 第二、三档不解锁；UI 只显示第一档入口 |
| 第一档存入但未满 | 允许追加；第二档解锁条件按"第一档存入满 1 天后"计时，而非满仓后 |
| "领取并重投"时精粹不足 | 允许部分重投：先将利息自动存入第一档，剩余利息留在账户 |
| 存入 0 | 不显示红点，不占注意力 |
| 超 2 天未登录 | 利息冻结（各档分别计最多 2 天），提示"金库满溢，增值已暂停，登录激活" |
| 账号同步失败 | 本地暂存各档状态，下次同步以服务器时间为准合并 |
| 精粹不足追加 | 提示当前余额 + 快捷入口跳转日常任务 |
| 月卡到期时有超限存入 | 保留原存入量，但下次重投时上限降回 2,000 |

## 八、与现有系统关系

| 系统 | 关系 |
|------|------|
| **兑换商店** | 金库产出精粹 → 玩家有更多换碎片的资源；金库不替代商店，而是给商店补货 |
| **签到系统** | 签到被动给，金库主动操作，互补 |
| **日常任务** | 任务获得精粹 → 存入金库 → 次日领取；形成"任务→存→取"的日常链 |
| **累计奖励** | 累计奖励是里程碑（一次性大量）；金库是日常循环（小量稳定），节奏不冲突 |
| **每日特惠** | 特惠付费获得精粹；金库是免费补充，不影响付费意愿 |

## 九、运营扩展

### 限时加息

节假日"金库加息"（全档位 ×1.5，持续 3 天）：
- 拉回流失玩家
- 配合节日礼包销售推送

### 新手加息

注册第 1-7 天丰厚档收益率提升至 25%（原 15%）：
- D2 关键期感知最强
- 7 天后自动恢复

### 月卡特权

月卡持有者存入上限提升至 **3,000 精粹**（原 2,000），同享 15% 收益率：

- 日最高产出：450 精粹/天
- 以"更大仓位"体现付费价值，而非另开档位，规则简单易懂

## 十、实现要点

### 数据结构

```lua
-- HeroData.activityData.vault
-- 每档独立记录，支持时序解锁逻辑
{
    tiers = {
        -- 第一档：随时可存，领取并重投后重置
        [1] = {
            deposit     = 2000,       -- 当前存入量
            depositedAt = 1745001600, -- 本次存入时间戳（用于判断次日解锁）
            pendingInterest = 300,    -- 待领取利息
            accruedDays = 1,          -- 已累计未领天数（>=2 停止增值）
        },
        -- 第二档：第一档存入满 1 天后解锁
        [2] = {
            deposit     = 600,
            depositedAt = 1745088000,
            pendingInterest = 48,
            accruedDays = 1,
        },
        -- 第三档：第二档存入满 1 天后解锁
        [3] = {
            deposit     = 0,
            depositedAt = nil,        -- nil 表示尚未解锁/存入
            pendingInterest = 0,
            accruedDays = 0,
        },
    },
    lastOpenTime = 1745088000,        -- 上次打开面板时间（用于前端防抖）
}
```

### 档位配置

```lua
local VAULT_TIERS = {
    { cap = 2000, rate = 0.15 },  -- 第一档：15%，最高 300/天
    { cap = 1000, rate = 0.08 },  -- 第二档：8%，最高 80/天
    { cap = 500,  rate = 0.04 },  -- 第三档：4%，最高 20/天
}

-- 档位解锁判断：上一档存入后是否已过 1 天
local function IsTierUnlocked(vault, tierIndex)
    if tierIndex == 1 then return true end
    local prev = vault.tiers[tierIndex - 1]
    if not prev or not prev.depositedAt then return false end
    return (os.time() - prev.depositedAt) >= 86400
end
```

### 利息计算

```lua
-- 单档日利息
local function CalcTierInterest(tierIndex, deposit)
    local cfg = VAULT_TIERS[tierIndex]
    if not cfg or deposit <= 0 then return 0 end
    return math.max(1, math.floor(math.min(deposit, cfg.cap) * cfg.rate))
end

-- 全金库日利息总计
local function CalcTotalDailyInterest(vault)
    local total = 0
    for i = 1, 3 do
        local t = vault.tiers[i]
        if t and t.deposit > 0 then
            total = total + CalcTierInterest(i, t.deposit)
        end
    end
    return total
end

-- CalcTierInterest(1, 2000) → 300
-- CalcTierInterest(2, 1000) → 80
-- CalcTierInterest(3, 500)  → 20
-- 三档全满合计 → 400
```

### 结算逻辑

```lua
-- 打开金库面板时触发，无需服务器定时任务
local function Settle(vault)
    local now = os.time()
    for i = 1, 3 do
        local t = vault.tiers[i]
        if t and t.deposit > 0 and t.depositedAt then
            local daysPassed = math.floor((now - t.depositedAt) / 86400)
            if daysPassed >= 1 then
                -- 最多累积 2 天
                local canAccrue = math.max(0, 2 - t.accruedDays)
                local effectiveDays = math.min(daysPassed, canAccrue)
                if effectiveDays > 0 then
                    t.pendingInterest = t.pendingInterest
                        + CalcTierInterest(i, t.deposit) * effectiveDays
                    t.accruedDays = t.accruedDays + effectiveDays
                end
            end
        end
    end
    vault.lastOpenTime = now
end
```

### 领取并重投

```lua
-- 一键全部领取并重新激活第一档
local function CollectAndReinvest(vault, playerCurrency)
    local totalInterest = 0
    for i = 1, 3 do
        local t = vault.tiers[i]
        totalInterest = totalInterest + (t.pendingInterest or 0)
    end
    -- 发放利息给玩家
    playerCurrency.shadow_essence = playerCurrency.shadow_essence + totalInterest
    -- 重置所有档位（本金归还给玩家总资产，重投第一档）
    local totalDeposit = 0
    for i = 1, 3 do
        totalDeposit = totalDeposit + (vault.tiers[i].deposit or 0)
    end
    -- 初始化归零
    for i = 1, 3 do
        vault.tiers[i] = { deposit = 0, depositedAt = nil, pendingInterest = 0, accruedDays = 0 }
    end
    -- 自动重投第一档（如果精粹足够）
    local reinvestAmount = math.min(totalDeposit + totalInterest,
        VAULT_TIERS[1].cap, playerCurrency.shadow_essence)
    vault.tiers[1] = { deposit = reinvestAmount, depositedAt = os.time(),
        pendingInterest = 0, accruedDays = 0 }
    playerCurrency.shadow_essence = playerCurrency.shadow_essence - reinvestAmount
end
```
