# 装备淬炼系统设计文档

> 暗黑召唤：无尽征途 — 装备淬炼（Tempering）系统
> 最后更新: 2026-04-09

---

## 一、系统概述

淬炼是装备系统的终局养成内容。当装备升至红色品质满级（Lv.4000）后，玩家可通过反复淬炼为装备附加额外的随机属性词条，提供更深层的数值成长空间。

**核心循环**: 消耗白玉 → 淬炼 → 40%概率提升属性 → 锁定好词条 → 继续淬炼

---

## 二、装备进阶费用（修订）

淬炼系统上线时，同步调整装备进阶（突破）的锻魂铁消耗：

| 品质 | 名称 | 等级段 | 进阶费用 (锻魂铁) | 变更 |
|------|------|--------|------------------|------|
| 绿 → 蓝 | 腐骨 → 冥铁 | Lv.1~200 | **200** | 原 0 → 200 |
| 蓝 → 紫 | 冥铁 → 噬魂 | Lv.201~1000 | **1,000** | 原 50 → 1,000 |
| 紫 → 橙 | 噬魂 → 渊狱 | Lv.1001~2000 | **2,700** | 原 200 → 2,700 |
| 橙 → 红 | 渊狱 → 灭世 | Lv.2001~3000 | **8,000** | 原 500 → 8,000 |
| 红 → 淬炼 | 灭世 Lv.4000 | Lv.3001~4000 | **2,000 暗影精粹** | 新增，开启淬炼 |

**单件装备全进阶累计**:

| 消耗项 | 数量 |
|--------|------|
| 锻魂铁 (绿→红) | 200 + 1,000 + 2,700 + 8,000 = **11,900** |
| 暗影精粹 (开启淬炼) | **2,000** |

**4 件装备全进阶累计**: 锻魂铁 47,600 + 暗影精粹 8,000

## 三、淬炼解锁条件

| 条件 | 说明 |
|------|------|
| 主线关卡 | 通过第 **3001** 关 (bestStage >= 3001) |
| 装备等级 | 该件装备已达 **Lv.4000**（红色满级）|
| 解锁费用 | **2,000 暗影精粹** (shadow_essence)，按件收取 |

> 解锁后该装备永久开启淬炼功能，无法再进行"一键重生"（重置）。
> 但可以无损转移给其他英雄。

## 四、淬炼机制

### 4.1 基础规则

| 参数 | 值 |
|------|---|
| 每次淬炼消耗 | **100 白玉** (pale_jade) |
| 成功率 | **40%** |
| 成功效果 | 随机刷新一个孔位的属性（类型+数值档位）|
| 失败效果 | 无变化，消耗不退还 |

### 4.2 孔位系统

每件装备最多 **5 个淬炼孔位**。初始解锁 1 个，后续通过累计淬炼次数解锁：

| 孔位编号 | 解锁条件 | 说明 |
|----------|---------|------|
| 第 1 孔 | 解锁淬炼时自动开启 | 免费 |
| 第 2 孔 | 累计淬炼 **10** 次 | |
| 第 3 孔 | 累计淬炼 **100** 次 | |
| 第 4 孔 | 累计淬炼 **1,000** 次 | |
| 第 5 孔 | 累计淬炼 **10,000** 次 | |

> "累计淬炼次数"包含成功和失败，按件独立统计。

### 4.3 淬炼属性池

本游戏是塔防游戏，英雄（塔）的核心能力是对敌人输出伤害，因此淬炼属性池聚焦于 **进攻类属性**，共 13 种：

#### 属性列表与最大加成

| # | 属性 | 代码 key | 最大加成 | 说明 |
|---|------|----------|---------|------|
| 1 | 攻击力 | `atk` | +10% | 提高塔的基础攻击 |
| 2 | 攻速 | `spd` | +10% | 提高塔的攻击速度 |
| 3 | 暴击率 | `critRate` | +20% | 提高暴击触发概率 |
| 4 | 暴击伤害 | `critDmg` | +30% | 提高暴击时的额外伤害倍率 |
| 5 | 破甲 | `armorPen` | +20% | 无视敌人部分护甲 |
| 6 | 伤害加成 | `dmgBonus` | +15% | 最终伤害乘算加成 |
| 7 | 技能伤害 | `skillDmg` | +20% | 技能释放时的额外伤害 |
| 8 | 控制命中 | `ctrlHit` | +20% | 提高控制技能（眩晕/减速等）的命中率 |
| 9 | 火元素伤害 | `elemFire` | +25% | 火属性英雄额外伤害 |
| 10 | 冰元素伤害 | `elemIce` | +25% | 冰属性英雄额外伤害 |
| 11 | 暗影伤害 | `elemShadow` | +25% | 暗影属性英雄额外伤害 |
| 12 | 毒元素伤害 | `elemPoison` | +25% | 毒属性英雄额外伤害 |
| 13 | 雷元素伤害 | `elemLightning` | +25% | 雷属性英雄额外伤害 |

> 13 种属性等概率出现 (每种 ≈7.69%)

#### 各部位"攻击力"词条规则

"攻击力"词条在不同部位上加成的是 **该部位自身属性**，数值 = 该部位 1 级基础值 × (50%~150%)，相当于多升 1 级的提升量（在档位内随机浮动）。

| 部位 | 属性类型 | 1级基础值 | 淬炼加成范围 (×50%~150%) | 说明 |
|------|---------|----------|------------------------|------|
| 武器 (weapon) | 攻击力 | 12 | +6~18 固定攻击 | 固定值加成 |
| 铠甲 (armor) | 伤害加成 | 0.002 | +0.1%~0.3% 伤害加成 | 百分比加成 |
| 头盔 (helmet) | 暴击伤害 | 0.003 | +0.15%~0.45% 暴击伤害 | 百分比加成 |
| 战马 (mount) | 元素伤害 | 0.002 | +0.1%~0.3% 元素伤害 | 百分比加成 |

> 公式: `加成值 = EQUIP_STAT_BASE[该部位stat] × rand(0.5, 1.5)`
> 档位颜色仍影响浮动范围：白档偏低(50%~80%)，红档偏高(130%~150%)

### 4.4 数值档位与颜色

属性数值分为 6 档（白/绿/蓝/紫/橙/红），每档加成按顺序递增，红色为最高档。

| 档位颜色 | 出现概率 | 数值范围（占最大值比例）|
|----------|---------|---------------------|
| 白 | **65.0%** | 1%~16% |
| 绿 | **20.0%** | 17%~33% |
| 蓝 | **11.0%** | 34%~50% |
| 紫 | **2.40%** | 51%~70% |
| 橙 | **1.20%** | 71%~90% |
| 红 | **0.40%** | 91%~100% |

**举例** — 暴击率（最大 20%）：

| 档位 | 数值范围 | 概率 |
|------|---------|------|
| 白 | +0.2%~3.2% | 65% |
| 绿 | +3.4%~6.6% | 20% |
| 蓝 | +6.8%~10.0% | 11% |
| 紫 | +10.2%~14.0% | 2.4% |
| 橙 | +14.2%~18.0% | 1.2% |
| 红 | +18.2%~20.0% | 0.4% |

**举例** — 武器攻击固定值（9~18）：

| 档位 | 数值范围 | 概率 |
|------|---------|------|
| 白 | +9 | 65% |
| 绿 | +10~11 | 20% |
| 蓝 | +12~13 | 11% |

## 五、孔位锁定系统

### 5.1 锁定规则

玩家可以锁定已有满意属性的孔位，使其在淬炼时不被刷新。

| 参数 | 值 |
|------|---|
| 锁定费用 | 每个锁定孔位消耗 **100 白玉** + **1 彩玉** (rainbow_jade) |
| 叠加规则 | 锁 N 个孔位，额外消耗 N 个彩玉（白玉固定 100）|
| 锁定效果 | 被锁定的孔位在淬炼时不会被刷新 |
| 解锁 | 随时可取消锁定，无消耗 |

### 5.2 锁定费用示例

| 锁定孔位数 | 白玉消耗 | 彩玉消耗 | 总计（每次淬炼）|
|-----------|---------|---------|---------------|
| 0 | 100 | 0 | 100 白玉 |
| 1 | 100 | 1 | 100 白玉 + 1 彩玉 |
| 2 | 100 | 2 | 100 白玉 + 2 彩玉 |
| 3 | 100 | 3 | 100 白玉 + 3 彩玉 |
| 4 | 100 | 4 | 100 白玉 + 4 彩玉 |

> 锁定 4 个孔位 = 只刷新第 5 个孔位（精准定向淬炼）

## 六、新增货币

淬炼系统引入 2 种新货币：

| 货币 | 代码 key | 用途 | 获取途径 |
|------|----------|------|---------|
| 白玉 | `pale_jade` | 淬炼基础消耗 | 主线掉落、挂机、资源副本、商店 |
| 彩玉 | `rainbow_jade` | 锁定孔位消耗 | 高难副本、限时活动、商店 |

> 淬炼解锁消耗的 **暗影精粹** (shadow_essence) 为现有货币，无需新增。

### 6.1 白玉 (pale_jade) 经济设计

**消耗速率估算**（全力淬炼，不锁定）:
- 每次 100 白玉，1 件装备 5 孔全开需 10,000 次
- 全孔全开消耗: 10,000 × 100 = **1,000,000 白玉**
- 4 件装备: **4,000,000 白玉**

**建议产出**:

| 渠道 | 产出量 |
|------|--------|
| 主线 BOSS 掉落 | 5~20 / BOSS (关卡 3001+) |
| 挂机收益 | 10 + floor(bestStage/10) /小时 |
| 资源副本 (新增白玉矿场) | 50~2000 /波 (复用资源副本框架) |
| 每日任务 | 200~500 /天 |

### 6.2 彩玉 (rainbow_jade) 经济设计

彩玉是稀缺货币，控制精准淬炼的节奏。

**建议产出**:

| 渠道 | 产出量 |
|------|--------|
| 试炼塔 (塔 30+) | 1~3 /通塔 |
| 资源副本 (第 20 波) | 1 /次 |
| 限时活动 | 5~10 /活动 |
| 商店 (暗影精粹兑换) | 500 精粹 = 1 彩玉 |

## 七、数值影响分析

### 7.1 淬炼对战力的影响

以武器为例，5 孔全部淬炼到最高档（红色）的理论极限：

| 孔位 | 最佳属性 | 红色档最大值 |
|------|---------|-------------|
| 1 | 攻击 | +6~18 固定攻击 |
| 2 | 暴击伤害 | +30% |
| 3 | 暴击率 | +20% |
| 4 | 破甲 | +20% |
| 5 | 伤害加成 | +15% |

> 实际上全红色概率极低: 0.4%^5 ≈ 0.00000001%（十亿分之一）
> 全紫以上的实用目标: 2.4%^5 ≈ 0.00008%（百万分之一），需要大量淬炼

### 7.2 期望淬炼次数

获得一个目标属性+目标档位的期望次数（13 种属性，每种 ≈7.69%）:

| 目标 | 概率 | 期望次数 | 期望白玉消耗 |
|------|------|---------|-------------|
| 任意属性·白档 | 40% × 65% = 26% | 3.8 次 | 380 |
| 指定属性·任意档 | 40% × 7.69% ≈ 3.08% | 33 次 | 3,300 |
| 指定属性·蓝档以上 | 40% × 7.69% × 15% ≈ 0.46% | 217 次 | 21,700 |
| 指定属性·紫档以上 | 40% × 7.69% × 4% ≈ 0.12% | 813 次 | 81,300 |
| 指定属性·红档 | 40% × 7.69% × 0.4% ≈ 0.012% | 8,125 次 | 812,500 |

> 淬炼是超长线养成系统，不需要快速毕业

## 八、数据结构设计

### 8.1 装备淬炼存档

```lua
-- HeroData.equips[heroId][slotId] 扩展字段
{
    level = 4000,
    tierIdx = 5,           -- 红色品质
    tempering = {          -- 淬炼数据（Lv.4000 解锁后才有）
        unlocked = true,   -- 是否已解锁
        totalAttempts = 0, -- 累计淬炼次数（含失败）
        slots = {
            -- 每个孔位: { attrId, value, tier, locked }
            [1] = { attrId = "critRate", value = 0.12, tier = "blue",  locked = false },
            [2] = { attrId = "critDmg",  value = 0.18, tier = "green", locked = true },
            [3] = nil,  -- 未解锁
            [4] = nil,
            [5] = nil,
        },
    },
}
```

## 十、与现有系统的关联

| 现有系统 | 关联方式 |
|----------|---------|
| 装备系统 (EquipData) | 淬炼属性叠加到 `GetTotalBonus()` 的计算中 |
